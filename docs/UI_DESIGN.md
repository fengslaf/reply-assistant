# UI 设计文档

> 基于用户反馈优化，版本 v1.1

## V0.2.04 当前界面

- 首页已经升级为 `快捷助手全家桶`，顶部展示版本和当前状态，中间放两个同高卡片：`回复助手` 与 `客户数据系统`。
- `回复助手` 卡片里的 `启动` 现在直接进入主界面，不再经过旧的登录页 / 本地启动页。
- `客户数据系统` 默认以表格形式展示，设置页负责导入、查看记录和数据整理。
- 界面控件以 Windows 原生的白底黑字普通按钮为主，避免第一次打开时出现蓝底白字或样式不统一。
- 首页右上角保留帮助、设置和回首页入口；关闭仍由系统窗口的 `X` 负责。

> 下面的“本地模式”布局草图保留为早期设计参考；当前实际首页请以上面的 V0.2.04 说明为准。

## 二、必须新增的功能

### 1. 设置窗口

位置：窗口右上角 ⚙️ 按钮

设置项：
| 设置项 | 说明 |
|--------|------|
| 快捷键设置 | 全局唤起快捷键，默认 Ctrl+Shift+Y |
| API Key 配置 | DeepSeek/OpenAI key，带测试连通性按钮 |
| 数据存储路径 | 显示当前目录，提供"打开文件夹"按钮 |
| 检索参数 | top_k 数量（默认 3）、相似度阈值 |

### 2. 帮助文档

位置：设置按钮旁边 ❓ 按钮

内容：
- 快速上手：三步走（复制问题 → 快捷键 → 选回复）
- 如何收藏回复
- 如何导入历史数据
- 标签体系说明
- 常见问题 FAQ

### 3. 状态栏

底部显示：
- 当前用户名
- 本地样本总数
- 数据存储路径

## 四、数据存储位置

默认路径：
- Windows: `%APPDATA%\QuickReplyAssistant\`
- macOS: `~/Library/Application Support/QuickReplyAssistant/`
- Linux: `~/.config/quickreplyassistant/`

存储内容：
- `samples.db` - SQLite 数据库
- `chroma/` - Chroma 向量库
- `config.json` - 配置文件（API key 等）

## 五、交互优化

### 1. 收藏样本弹窗
点击"收藏样本"后弹出：
- 场景标签选择（下拉框）
- 备注输入（可选）
- 确认/取消按钮

### 2. 检索自动触发
快捷键唤起时自动：
- 从剪贴板填充问题
- 自动触发检索

### 3. Loading 状态
调用 API 时显示"正在生成..."状态
